378. Kth Smallest Element in a Sorted Matrix

Posted on 2021-03-11 In algorithm

Method 1

Using a priority queue to keep a min heap of min(N,k). We need to define our own heap node to keep the information about its row, column and value. We loop for k times, each time we pop the top element and push into the queue the element after it.

Method 2

The solution is implemented using binary search but is in a way different than typical one. Here we take the middle one as the average of start and end.

  • We search through the matrix to get the number of elements smaller or equal to mid. At the same time, we keep track of the biggest number smaller than or equal to mid and smallest number larger than mid. We keep updating the end and start until the count is equal to k.

371. Sum of Two Integers

Posted on 2021-03-11 In algorithm

First we need to turn both a and b to positive and we need to make sure that a>b. If the signs of a and b are different, then it is considered to be a-b; If identical, then a+b.

  • For a+b, we can get the result by manipulation of its sum without carry and carry. We get the former by XOR and latter by (a&b)<<1.
  • For a-b, we can get the result by manipulation of its sum without carry and carry. We get the former by XOR and latter by (~a&b)>>1.

289. Game of Life

Posted on 2021-03-06 Edited on 2021-03-07 In algorithm

The most important trick of this problem(to do it inplace) is to use different integers to denote live->dead and dead->live so that they won’t corrupt the initial values. In my solution, I use -1 and 2 to represent them respectively.

  • Follow up:

    • It would be computationally impossible to iterate a matrix that large.

    • It would not be possible to store that big a matrix entirely in memory. We have huge memory capacities these days i.e. of the order of hundreds of GBs. However, it still wouldn’t be enough to store such a large matrix in memory.

    • We would be wasting a lot of space if such a huge board only has a few live cells and the rest of them are all dead. In such a case, we have an extremely sparse matrix and it wouldn’t make sense to save the board as a “matrix”.

    • Solution:
      For the first two situations, we read one row at a time from the file and at max we will have 3 rows in memory. We will keep discarding rows that are processed and then we will keep reading new rows from the file, one at a time.
      For the third situation, we could only keep the set of the coordinates of live cells, and count the times of the appearance of its surrounding cells, use it to update the board.
