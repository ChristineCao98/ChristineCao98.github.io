140. Word Break II

Posted on 2021-05-08 In algorithm

Initially the approach I took is to keep a hashmap with the key being the index and value the list of all the concatenations form begin to that index. But it is very costly. The rationale is twofold:

  1. we have to keep many copies of string in the memory.
  2. we have to iterate through the string to make a new copy, which is time-consuming.
    To optimize it, we can keep the index rather than the string and do a backtracking to get the result afterwards.

42. Trapping Rain Water

Posted on 2021-05-03 In algorithm

For each grid i the water it can hold is min(leftmax[i],rightmax[i])-h[i].
leftmax[i]=max(h[0],h[1],,h[i-1])
rightmax[i]=max(h[i+1],h[i+2],h[length-1])
It can be illustrated in the diagram below:

The gist is leftmax[i-1]<=leftmax[i], so if rightmax[i]<leftmax[i-1], rightmax[i]<leftmax[i]. It is same for the case of leftmax[i]<=rightmax[i+1]
